Các khái niệm về Spring (Phần này dài ấy ,ông nào lười đọc thì nghe giảng cho nhanh)

  1. Java Spring là gì ?

    Lưu ý: đừng ai đi gg dịch mấy cái từ tiếng anh trong thư viện.Vì nó không mang ý nghĩa gì đâu.

    1. Java Spring là một "framework" sử dụng ngôn ngữ Java.

      => để học Spring, chúng ta "bắt buộc" cần biết "cú pháp của java"

      Khi nó là framework, nó thưởng "huge" (to lớn), và cung cấp bộ khung/hạ tầng để chúng ta tạo dự án/code nhanh hơn.
    2. Spring Boot là gì ?

      Spring Boot = Spring + Boot Boot === Bootstrapping app (chạy dự án lên/start app)

      Mục đích
      1. - Làm đơn giản hóa quá trình xây dựng Java app sử dụng Spring framework
      2. - Giảm thiểu tối đa quá trình "cấu hình" cho dự án:
      3. + đơn giản hóa quá trình cài đặt thư viện, quản lý cấu hình thông qua maven...
      4. - cung cấp sẵn server để chạy ứng dụng (embedded HTTP server)
      5. không cần phải cài đặt server riêng lẻ (chương trước chạy có cần setup gì đâu :v)

    Khái niệm chỉ thế thôi :v nói thêm các ông cũng chả nhớ để làm gì

  2. Cấu trúc dự án Spring

    Như ở chương trước ta đã Hello World với Spring nên hôm nay ta sẽ đi vào phân tích cấu trúc thư mục trong Spring (tớ sẽ chỉ đề cập những file chúng ta sẽ thao tác thôi nhé!)

      Các thư mục viết code
    1. - src : thư mục chứa code “chính” của dự án
    2. resources : chứa nguồn tài nguyên của dự án như frontend (html, css, js, images), file cấu hình…
    3. test: thư mục viết code test(chả bao h động :v)
    4. target: mã nguồn java sau khi được dịch (JVM sẽ chạy thư mục target)
      Maven và Gradle ? (Trong khoá học này ta sủ dụng Maven)
    1. hiểu 1 cách đơn giản: giúp build dự án cho java
    2. về cách sử dụng thì như nhau thôi : lên mạng tìm > import dependencies > Dùng >Hết.
    3. (ưu hay nhược thì sau này phải vào đúng bài toán của chúng ta mới biết được)
    4. Trong dự án của spring có 2 file:
      1. mvnw => dành cho macos
      2. mvnw.cmd => dành cho windows
      3. => nhờ có 2 file này, chúng ta có thể sử dụng các câu lệnh của maven mà không cần cài đặt thủ công (đã test ở chương trước )

    5. pom.xml (với Maven), build.gradle.kts (với Gradle)

      1. - sử dụng để xử lý project (compile/build & run)
  3. Kiến trúc của Spring Framework

    1. Tài liệu :link

    2. với nội dung khoá này ,mình sẽ không học hết trên tài liệu đâu nhưng tớ sẽ điểm qua những ý chính:
        1. spring container:
        1. (đọc cái biết luôn): là thùng chứa(gg dịch) hay là một thành phần cốt lõi của Spring Framework ( một thôi nhé không phải all đâu),
        2. Quản lý tất cả các Bean
        3. Container thực hiện việc cung cấp các phụ thuộc cho bean, giúp các bean không cần tự mình tạo ra các đối tượng phụ thuộc. hay còn gọi là Dependency Injection (DI)
        4. Inversion of Control (IoC) IoC là nguyên tắc mà Container chịu trách nhiệm điều khiển luồng ứng dụng và quản lý các đối tượng, thay vì để chúng tự quản lý.
      2. Bean là bất kỳ đối tượng nào được định nghĩa trong Spring Container thông qua file cấu hình XML, Java configuration, hoặc annotation. Spring Container kiểm soát toàn bộ vòng đời của các bean.(hiểu nhanh là class trong Java ấy v:)
      3. Dependency Injection (nghe giảng nhé :v)
      4. Annotation (chú thích), là cách chúng ta “trang trí” class của Java. (decorator) Mục đích là “tăng thêm sức mạnh” cho class đấy (meta-data)
