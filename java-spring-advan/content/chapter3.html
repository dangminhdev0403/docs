Kiến thức cần biết trước khi code Spring

Cái này tớ cần phải làm trước khi mọi người tiếp xúc với Spring.

Cho đỡ bỡ ngỡ

  1. Funcfional Interfaces: là một interface chỉ chứa một phương thức trừu tượng duy nhất. Nó có thể có các phương thức mặc định hoặc static, nhưng chỉ được phép có một phương thức trừu tượng. Functional Interface giúp hỗ trợ lập trình hàm (functional programming) trong Java, đặc biệt là khi sử dụng Lambda Expressions.
  2. Lambda Expression (biểu thức lambda) là một tính năng được giới thiệu từ Java 8, giúp viết code ngắn gọn hơn khi làm việc với Functional Interface. Lambda cho phép viết các phương thức dưới dạng hàm mà không cần khai báo lớp hoặc phương thức thông thường.


  3. Method Reference là một cách viết ngắn gọn hơn của Lambda Expression, dùng để tham chiếu đến một phương thức đã tồn tại mà không cần gọi trực tiếp. Nó giúp code dễ đọc hơn, đặc biệt khi lambda chỉ gọi một phương thức duy nhất.

    Cái ví này hơi vô dụng tí , mình sẽ cho cái khác

  4. Default Methods trong Java là các phương thức có phần thân (body) bên trong interface.
  5. Stream API trong Java là một công cụ mạnh mẽ để xử lý dữ liệu theo kiểu functional programming. Nó giúp duyệt, lọc, biến đổi, và thu thập dữ liệu một cách dễ dàng và hiệu quả.

    ✔ Ghi nhớ: Stream không thay đổi dữ liệu gốc mà tạo luồng dữ liệu mới!

  6. Optional <T> là một lớp trong gói java.util được giới thiệu từ Java 8 để xử lý các giá trị có thể là null một cách an toàn, tránh lỗi NullPointerException.

    Full Phương thức trong Optional

    Phương thức Mô tả Ví dụ
    of(T value) Tạo một Optional chứa giá trị (không được null). Optional<String> opt = Optional.of("Hello");
    ofNullable(T value) Tạo một Optional, cho phép giá trị null. Optional<String> opt = Optional.ofNullable(null);
    empty() Trả về một Optional rỗng. Optional<String> opt = Optional.empty();
    isPresent() Kiểm tra xem có giá trị hay không. if (opt.isPresent()) { System.out.println(opt.get()); }
    ifPresent(Consumer<T> action) Thực hiện hành động nếu có giá trị. opt.ifPresent(System.out::println);
    ifPresentOrElse(Consumer<T> action, Runnable emptyAction) Thực hiện hành động nếu có giá trị, nếu không thì chạy emptyAction. opt.ifPresentOrElse(System.out::println, () -> System.out.println("No value"));
    get() Lấy giá trị bên trong Optional, nếu rỗng sẽ ném NoSuchElementException. String value = opt.get();
    orElse(T other) Trả về giá trị nếu có, nếu không trả về giá trị mặc định. String value = opt.orElse("Default Value");
    orElseGet(Supplier<T> supplier) Trả về giá trị nếu có, nếu không lấy giá trị từ Supplier. String value = opt.orElseGet(() -> "Generated Default Value");
    orElseThrow(Supplier<X> exceptionSupplier) Trả về giá trị nếu có, nếu không ném exception. String value = opt.orElseThrow(() -> new IllegalArgumentException("No value present"));
    map(Function<T, R> mapper) Chuyển đổi giá trị nếu có. Optional<Integer> length = opt.map(String::length);
    flatMap(Function<T, Optional<R>> mapper) Tương tự map() nhưng trả về Optional. Optional<String> upperOpt = opt.flatMap(s -> Optional.of(s.toUpperCase()));
    filter(Predicate<T> predicate) Lọc giá trị theo điều kiện. Optional<String> filtered = opt.filter(s -> s.startsWith("H"));
