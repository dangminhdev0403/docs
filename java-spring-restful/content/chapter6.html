  7. Cách Read/Write Data

    Với database, chúng ta có 2 thao tác : READ (đọc dữ liệu) và WRITE (update/insert/delete), tức là ghi/sửa đổi dữ liệu

    1. cơ sở dữ liệu quan hệ (relational database)
    2. cơ sở dữ liệu phi quan hệ (no sql)
    Java là một ngôn ngữ hướng đối tượng, nên mình lựa chọn cơ sở dữ liệu quan hệ (relational database)

      Để thực hiện điều trên, chúng ta có 2 cách:
    1. Sử dụng JDBC( Java Database Connectivity):Là cách chúng ta kiểm soát việc sử dụng database thông qua câu lệnh truy vấn, ví dụ truy vấn SQL
    2. Sử dụng ORM(Object Relational Mapping):Là cách chúng ta truy cập/sử dụng database thông qua các object định nghĩa trong code
  8. Hibernate vs Spring Data JPA tham khảo: youtube , docs
    1. Hibernate là một framework ORM (Object-Relational Mapping) trong Java. Nó giúp ánh xạ các đối tượng Java vào các bảng trong cơ sở dữ liệu quan hệ. Hibernate cung cấp các tính năng mạnh mẽ để thực hiện các thao tác CRUD, truy vấn, và quản lý các đối tượng trong cơ sở dữ liệu.
    2. Spring Data JPA là một phần của Spring Framework, giúp trừu tượng hóa việc sử dụng JPA (Java Persistence API) để tương tác với cơ sở dữ liệu. Spring Data JPA sử dụng Hibernate hoặc bất kỳ implementation JPA nào khác (như EclipseLink, OpenJPA) để thực hiện các thao tác dữ liệu.

    9. Ở trong khoá này (và cả khoá sau nếu có) mình sẽ sử dụng cả 2 luôn nhé !
