Restful API với Spring

Trước tiên mọi người lên mạng tải và đăng kí -> đăng nhập Postman về nhé.

Trong chương này mình sẽ liệt kê toàn bộ kiến thức chúng ta phải nắm rõ với mô hình này.

  1. JSON(JavaScript Object Notation). tài liệu tại Đây
    • Khi dịch JSON ra ta thấy ngay từ khoá Object tức là đối tượng hay class trong Java.
    • với javascripts: JSON là một kiểu dữ liệu dạng text hay một chuỗi kĩ tự(string) ví dụ :

      '{"name":"John", "age":30, "car":null}'
    • => 2 ngôn ngữ này (ở đây là java và javascripts) sẽ nói chuyện với nhau thông qua JSON.Nói chuyện như nào , phải đi code mới rõ được :v
    • Nhắc lại : ai học yếu hướng đối tượng (OOP) thì nên nắm chắc kiến thức rồi hãy học món này .
  2. HTTPS tại liệu tại Đây

    Hiểu đơn giản HTTP là công cụ để giúp ta có thể truy cập data trên internet.

    Mình sẽ ví dụ với các thao tác, thêm, sửa, xoá: client giao tiếp với server thông qua các HTTP Method(GET ,PUT,POST,DELETE,...)

    Cấu trúc HTTP Request:(Client gửi lên server)

    1. Request line: method + url
    2. Header variable
    3. Message: JSON

    Cấu trúc HTTP Response:(server trả về cho client )

    1. version of the HTTP
    2. Status code + status message
    3. Message: JSON

  3. Api. tài liệu tại Đây
    1. Là link url ta định nghĩa tại backend
    2. Fontend sẽ dùng url đó để gọi yêu cầu lên backend
  4. Mô hình RESTful API (Representational State Transfer) là một kiểu kiến trúc phần mềm cho các dịch vụ web, trong đó các tài nguyên (resources) được xác định thông qua URL và thao tác với các tài nguyên này được thực hiện thông qua các phương thức HTTP như GET, POST, PUT, DELETE, v.v.
